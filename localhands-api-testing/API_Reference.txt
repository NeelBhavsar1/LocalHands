LOCALHANDS API REFERENCE
========================

A comprehensive reference for the LocalHands backend API endpoints.


BASE URL
--------
http://localhost:8080


AUTHENTICATION
--------------

The API uses JWT Token-based authentication via HTTP-only cookies:

- Access Token: Stored in 'accessToken' cookie, short-lived, used for API requests
- Refresh Token: Stored in 'refreshToken' cookie, long-lived, used to obtain new access tokens

For authenticated endpoints, the browser automatically sends these cookies. No manual header configuration is required for cookie-based requests.

Roles:
- USER: Standard authenticated user
- SELLER: User with service provider privileges (can create/modify listings)


================================================================================
AUTHENTICATION ENDPOINTS
================================================================================

Base path: /api/auth


POST /api/auth/register
-----------------------
Register a new user account.

Authentication: Public
Headers: None required

Body (JSON):
{
  "firstName": "string",
  "lastName": "string",
  "dateOfBirth": "YYYY-MM-DD",
  "email": "string",
  "password": "string",
  "serviceProvider": boolean,
  "rememberMe": boolean
}

Success (200):
"Registered and logged in to new account."

Response includes Set-Cookie headers for accessToken and refreshToken.

Common Errors:
- 400 Bad Request: Invalid input data
- 409 Conflict: Email already registered


POST /api/auth/login
--------------------
Authenticate and log in an existing user.

Authentication: Public
Headers: None required

Body (JSON):
{
  "email": "string",
  "password": "string",
  "rememberMe": boolean
}

Success (200):
"Logged in to account."

Response includes Set-Cookie headers for accessToken and refreshToken.

Common Errors:
- 401 Unauthorized: Invalid credentials


POST /api/auth/logout
---------------------
Log out the current user and invalidate tokens.

Authentication: Public (cookies optional)
Headers: None required
Body: None

Success (200):
"Logged out successfully."

Response includes Set-Cookie headers to clear accessToken and refreshToken.


POST /api/auth/refresh
----------------------
Refresh the access token using the refresh token.

Authentication: Public (requires refreshToken cookie)
Headers: None required

Cookies Required:
- refreshToken: The refresh token from previous login

Body: None

Success (200):
"Created new refresh and access tokens."

Response includes Set-Cookie headers with new tokens.

Common Errors:
- 401 Unauthorized: Refresh token missing or invalid


GET /api/auth/confirm-email
---------------------------
Confirm user's email address using token from email.

Authentication: Public
Headers: None required

Query Params:
- token (string, required): Email confirmation token

Success (200):
"Email confirmed successfully."


GET /api/auth/activate-account
--------------------------------
Activate a deactivated user account.

Authentication: Public
Headers: None required

Query Params:
- token (string, required): Account activation token

Success (200):
"Account activated successfully."


GET /api/auth/deactivate-account
--------------------------------
Deactivate the current user account.

Authentication: Public (requires token)
Headers: None required

Query Params:
- token (string, required): Account deactivation token

Success (200):
"Account deactivated successfully."

Response includes Set-Cookie headers to clear tokens.


POST /api/auth/forgot-password
------------------------------
Request a password reset email.

Authentication: Public
Headers: None required

Query Params:
- email (string, required): User's email address

Success (200):
"Successfully sent password reset code to {email}"


POST /api/auth/verify-password-reset-code
-----------------------------------------
Verify the password reset code sent via email.

Authentication: Public
Headers: None required

Body (JSON):
{
  "email": "string",
  "code": "string"
}

Success (200):
"reset-token-string"

Common Errors:
- 400 Bad Request: Invalid code


POST /api/auth/reset-password
-----------------------------
Reset password using verified token.

Authentication: Public
Headers: None required

Body (JSON):
{
  "email": "string",
  "token": "string",
  "password": "string"
}

Success (200):
"Successfully set new password."

LISTING ENDPOINTS
================================================================================

Base path: /api/listings


POST /api/listings
------------------
Create a new service listing.

Authentication: Required (SELLER role only)
Headers:
- Cookie: accessToken and refreshToken
- Content-Type: multipart/form-data

Body (Multipart):
- listing (JSON): Listing data
  {
    "name": "string",
    "description": "string",
    "latitude": 0.0,
    "longitude": 0.0,
    "categoryIds": [1, 2, 3]
  }
- photos (List of Files): Listing images
- altTexts (List of Strings): Alt text for each photo

Success (200):
{
  "listingId": 1,
  "name": "string",
  "description": "string",
  "latitude": 0.0,
  "longitude": 0.0,
  "creationTime": "2024-01-01T00:00:00Z",
  "updatedTime": "2024-01-01T00:00:00Z",
  "categories": [
    {
      "id": 1,
      "category": "string"
    }
  ],
  "photos": [
    {
      "id": 1,
      "url": "string",
      "altText": "string"
    }
  ],
  "reviews": [...],
  "seller": {
    "id": 1,
    "firstName": "string",
    "lastName": "string",
    "profilePhoto": {...}
  }
}

Common Errors:
- 403 Forbidden: User does not have SELLER role

Success (200):
[
  {
    "listingId": 1,

  }

Get all available listing categories.

Authentication: Public

[
  {
    "id": 1,
    "category": "Plumbing"
  },
  {
    "id": 2,
    "category": "Electrical"

PUT /api/listings
-----------------
Update an existing listing.

Authentication: Required (SELLER role only)
Headers:
- Cookie: accessToken and refreshToken
- Content-Type: multipart/form-data

Query Params:
- listingId (long, required): ID of listing to update

Body (Multipart):
- listing (JSON): Updated listing data
  {
    "name": "string",
    "description": "string",
    "latitude": 0.0,
    "longitude": 0.0,
    "categoryIds": [1, 2]
  }
- photos (List of Files): Updated listing images
- altTexts (List of Strings): Alt text for each photo

Success (200):
{
  "listingId": 1,
  "name": "string",
  ...
}

Common Errors:
- 403 Forbidden: Not the listing owner or not a SELLER
- 404 Not Found: Listing does not exist

Authentication: Required (SELLER role only)
Headers:
- Cookie: accessToken and refreshToken

Query Params:
- listingId (long, required): ID of listing to delete

"Listing deleted successfully."

MESSAGE ENDPOINTS
================================================================================

Base path: /api/messages


POST /api/messages
------------------
Send a message to another user about a listing.

Authentication: Required (Authenticated User)
Headers:
- Cookie: accessToken and refreshToken
- Content-Type: application/json

Body (JSON):
{
  "recipientId": 1,
  "listingId": 1,
  "message": "string"
}

Success (200):
{
  "id": 1,
  "message": "string",
  "sentTime": "2024-01-01T00:00:00Z",
  "senderId": 1,
  "recipientId": 2
}


GET /api/messages/conversation
------------------------------
Get conversation messages between current user and another user about a listing.

Authentication: Required (Authenticated User)
Headers:
- Cookie: accessToken and refreshToken

Query Params:
- otherUserId (long, required): ID of the other conversation participant
- listingId (long, required): ID of the related listing

Success (200):
{
  "listingTitle": "string",
  "otherUserName": "string",
  "conversation": [
    {
      "id": 1,
      "message": "string",
      "sentTime": "2024-01-01T00:00:00Z",
      "senderId": 1,
      "recipientId": 2
    }
  ],
  "canSendMessages": boolean,
  "canAccessListing": boolean,
  "otherUserProfilePicture": "string"
}


GET /api/messages/inbox
-----------------------
Get all conversation previews for the current user's inbox.

Authentication: Required (Authenticated User)
Headers:
- Cookie: accessToken and refreshToken

Body: None

Success (200):
[
  {
    "listingId": 1,
    "listingTitle": "string",
    "otherUserId": 2,
    "otherUserName": "string",
    "lastMessage": "string",
    "lastMessageTime": "2024-01-01T00:00:00Z",
    "otherUserProfilePicture": "string"
  }
]

COOKIE SECURITY
================================================================================

All authentication cookies are:
- HttpOnly: Not accessible via JavaScript
- Secure: Only sent over HTTPS in production
- SameSite: Configured to prevent CSRF attacks
