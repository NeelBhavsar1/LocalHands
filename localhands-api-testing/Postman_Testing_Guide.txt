================================================================================
POSTMAN SETUP
================================================================================

STEP 1: Create Environment
--------------------------
1. Click "Environments" in left sidebar
2. Click "+" to create new environment
3. Name it "LocalHands Local"
4. Add these variables:

   Variable              Initial Value
   -----------------     -----------------
   baseUrl               http://localhost:8080
   accessToken           (leave blank)
   refreshToken          (leave blank)
   testEmail             test@example.com
   testPassword          TestPass123!
   testUserId            (leave blank)
   testListingId         (leave blank)

5. Click "Save"


STEP 2: Configure Cookie Handling
---------------------------------
1. Go to File > Settings > General
2. Ensure "Automatically follow redirects" is ON
3. In Tests tab, you can capture cookies from responses

================================================================================
COOKIE HANDLING IN POSTMAN
================================================================================

Manual Cookie Method:
---------------------
1. After login/register, check the response "Headers" tab
2. Look for "Set-Cookie" headers
3. Copy the values for accessToken and refreshToken
4. Go to your subsequent request's "Headers" tab
5. Add header: Cookie: accessToken=value; refreshToken=value


================================================================================
MULTIPART FORM-DATA IN POSTMAN
================================================================================

For file uploads (listings, profile photos):

1. Select "Body" tab
2. Select "form-data"
3. Add fields:
   - Key: "listing", Value: [paste JSON], Content-Type: application/json
   - Key: "photos", Value: [select "File" from dropdown, choose file]
   - Key: "altTexts", Value: "Main photo"
   - Key: "altTexts", Value: "Secondary photo" (add same key multiple times)

================================================================================
TROUBLESHOOTING
================================================================================

401 Unauthorized:
-----------------
- Check cookies are being sent
- Try refreshing token: POST /api/auth/refresh
- Login again to get fresh tokens

403 Forbidden:
--------------
- User doesn't have SELLER role (set serviceProvider=true in registration)
- Not the owner of the resource

404 Not Found:
--------------
- Check the ID in URL parameter
- Resource may have been deleted

400 Bad Request:
----------------
- Check JSON syntax
- Ensure required fields are present
- For multipart: check file types and sizes (<10MB)

500 Internal Server Error:
--------------------------
- Check server logs
- Verify database connection
- Check file upload limits
